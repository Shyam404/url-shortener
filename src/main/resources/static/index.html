How This URL Shortener Works

This system generates fixed-length (6 character) short codes using a hash-based approach combined with Base62 encoding.

  • The submitted long URL is hashed using a secure hashing algorithm (MessageDigest).
  • The hash bytes are converted into a positive numeric value.
  • The numeric value is encoded into Base62 (0-9, a-z, A-Z).
  • A fixed-length short code is extracted.
  • A unique database constraint ensures collision safety.
  • If a collision occurs, a salted retry mechanism regenerates a new code.
  • Redirection is handled using HTTP 302 to enable accurate click analytics.

The system is deployed on Render with PostgreSQL (Neon) and implements optimistic collision handling with database-level uniqueness guarantees.
